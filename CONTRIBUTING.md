# Contributing

1. Fork and branch from `main`.
2. Run `npm run lint`, `npm run test`, and `npm run build` before opening a PR.
3. Keep changes focused; match existing TypeScript and R3F patterns.
4. Do not commit large binary assets to Git; document them in README instead.

## Branches

Prefer `feature/…` or `fix/…` prefixes branched from `main`.

## Commits

Small, focused commits are easier to review than one large diff.
